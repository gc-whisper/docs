# GameRamp API Documentation

The GameRamp API provides programmatic access to track player events and retrieve personalized configurations and predictions.

## Base URL

All API requests should be made to:

```
https://api.gameramp.com
```

## API Categories

### Authentication APIs
APIs for managing authentication and tokens:

- **[Login](/api-reference/authentication)** - Authenticate and obtain JWT tokens
- **[Refresh Token](/api-reference/endpoints/auth/refresh)** - Refresh expired access tokens
- **[Logout](/api-reference/endpoints/auth/logout)** - Invalidate active tokens

### Data Transfer APIs
APIs for sending data to GameRamp:

- **[Events](/api-reference/endpoints/events)** - Track player events and behaviors

### Configuration APIs
APIs for retrieving predictions and configurations:

- **[Player Status](/api-reference/endpoints/player/status)** - Get player predictions (e.g., isPayer)
- **[Store Configs](/api-reference/endpoints/player/store-configs)** - Get personalized store products

## Authentication

All API endpoints require authentication using JWT tokens obtained through the login endpoint.

<Steps>
  <Step title="Obtain Credentials">
    Get your API Key and Secret from GameRamp
  </Step>
  <Step title="Generate Signature">
    Create HMAC-SHA256 signature with your Secret
  </Step>
  <Step title="Login">
    Exchange signature for JWT tokens
  </Step>
  <Step title="Make API Calls">
    Use JWT token in Authorization header
  </Step>
</Steps>

## Quick Start

<CodeGroup>
```bash Login
curl -X POST 'https://api.gameramp.com/auth/login' \
  -H 'Content-Type: application/json' \
  -d '{
    "apiKey": "your_api_key",
    "deviceId": "unique_device_id",
    "timestamp": "2025-01-15T10:00:00Z",
    "signature": "base64_encoded_signature"
  }'
```

```bash Use Token
curl -X GET 'https://api.gameramp.com/v1/player/status' \
  -H 'Authorization: Bearer YOUR_JWT_TOKEN' \
  -H 'X-Tenant-ID: your_tenant_id' \
  -H 'X-Game-Id: your_game_id' \
  -H 'X-Player-Id: player_123'
```
</CodeGroup>

## Response Format

All API responses follow a consistent JSON format:

### Success Response
```json
{
  "data": { ... },
  "status": "success"
}
```

### Error Response
```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human readable error message"
  },
  "status": "error"
}
```
